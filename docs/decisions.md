# Marginalia — Product Decisions

This document records important founder decisions.

---

# Decision 001

Guests can browse before signing up.

Reason:

People should understand the value of Marginalia before being asked to create an account.

---

# Decision 002

Book pages open with the most-loved passage.

Reason:

Marginalia is about conversations around books, not book summaries.

---

# Decision 003

The product uses its own vocabulary.

Examples:

* Reader Notes → In the Margins
* Save → Keep

Reason:

Products become memorable when they have their own language.

---

# Decision 004

No advertisements.

Reason:

Reading deserves an uninterrupted environment.

---

# Decision 005

Every feature must answer one question:

"Does this improve the reading experience?"

If not, it doesn't belong.
